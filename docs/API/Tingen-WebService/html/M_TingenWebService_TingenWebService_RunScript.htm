      Remarks

        Please read

      Important information about this method:
      • This method is required by Avatar and should not be modified.
      • Trace Logs can't go here because the logging infrastructure
        hasn't been initialized yet.

        Critical failure warning

      If Avatar doesn't pass an OptionObject or Script Parameter, the Tingen Web Service will fail.

      Since this can be difficult to troubleshoot, a Defcon Log will be created.

      Generally the fix for a missing OptionObject and/or Script Parameter is to re-import the Tingen Web Service WSDL from within Avatar.

      The main functions of this method are:

      • Receive the SentOptionObject and ScriptParameter from Avatar
      • Handle prototype requests
      • Handle standard requests

      When the Tingen Web Service receives a request from Avatar, the first thing it
      does is determine if it is a prototype or standard request.

      Prototype requests are used to test new functionality. They start with _p, and
      are handed off to Module.Prototype.

      Standard requests follow the normal Avatar workflow, which is:

      1. Initialize a new Tingen Session
      2. Start the Tingen Web Service
      3. Pass the SentOptionObject and ScriptParameter to Outpost31
      4. Outpost31 processes the request and returns a finalized OptionObject
      5. Return the finalized OptionObject to Avatar
