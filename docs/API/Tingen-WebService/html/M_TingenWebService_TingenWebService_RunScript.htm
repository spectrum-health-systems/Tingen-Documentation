      Remarks

      • This method is required by Avatar and should not be modified.
      • This is the main entry point for the Tingen Web Service, and it's where Avatar
        sends its requests.
      • Trace Logs can't go here because the logging infrastructure hasn't been
        initialized yet.

      The main components of this method are:

      • Receive the SentOptionObject and ScriptParameter from Avatar
      • Determine if the OptionObject and/or Script Parameter are missing,
        which is a critical failure
      • Handle prototype code
      • Handle requests sent from Avatar

      Prototype code is used to develop and test future functionality, and is handled
      differently than production code. Prototype code is handed off to Module.Prototype,
      and follows the logic of individual prototype commands instead of initializing a new
      session.
      Production code follows the standard Tingen Web Service logic, which looks like
      this:

      1. Initialize a new Tingen Session
      2. Start the Tingen Web Service
      3. Passing the SentOptionObject and ScriptParameter to Outpost31
      4. Outpost31 processes the request and returns a finalized OptionObject
      5. Returning the finalized OptionObject to Avatar
