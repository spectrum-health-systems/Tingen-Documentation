      Remarks

        About this property

      The version number is pulled from Properties/AssemblyInfo.cs.

      Version syntax
      The Tingen Web Service is closer to Semantic versioning than the Microsoft
      versioning scheme, and consists of up to three components:

      • The release year
      • The release month
      • An optional patch number
      Put those together and you get a version number that looks like this:
      Example version
      YY.MM.Patch (e.g., 25.02.1)
      This isn't 'Nam, there are rules:
      • The YY component is two digits (e.g., 2025 would be "25")
      • The MM component is two digits (e.g., July would be "07")
      • The Patch component is a single digit, and is 0 if there are no patches
      • The Revision component in the AssemblyInfo.cs file) is always 0

      For each release of Tingen the following lines:

      Before modification
      [assembly: AssemblyVersion("Major.Minor.Build.Revision")]
[assembly: AssemblyFileVersion("Major.Minor.Build.Revision")]
      Need to be modified as so:
      After modification
      [assembly: AssemblyVersion("YY.MM.Patch.0")]
[assembly: AssemblyFileVersion("YY.MM.Patch.0")]

      Example

      The December 2024 release, without a patch, would look like:
      Example AssemblyInfo.cs content
      [assembly: AssemblyVersion("24.12.0.0")]
[assembly: AssemblyFileVersion("24.12.0.0")]
