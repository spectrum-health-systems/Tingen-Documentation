      Remarks

        Important information about this method

      Before an OptionObject can be returned to Avatar, we need to make sure
      that all of the required OptionObject components exist and/or are
      modified correctly.

      This is done by calling the OptionObject's built
      in ToReturnOptionObject() method on the OptionObject, then
      sending it back to Avatar.

      Error Codes
      When an OptionObject is returned to Avatar, it requires one of the following
      single-digit integer Error Codes:

      Error CodeDescription
      0No action taken
      1 Returns an Error Message with an "Ok" button, and stops
      further processing of scripts
      2 Returns an Error Message with "OK" and "Cancel" buttons,
      and stops further scripts the user clicks "Cancel")
      3Returns an Error Message with "Ok" button
      4 Returns an Error Message with "Yes" and "No" buttons
      and stops further scripts the user clicks "No")
      5Returns a URL to be opened in a new browser
      6Returns a form to be opened in Avatar

        Common Error Codes

      The most common error codes are:
      • 1 – Used when you don't want the user to take an action, such as
        submitting a form
      • 3 – Used when you would like to notify the user of something
      • 4 – Used when you want warn the user they may want to make
        changes, and give them the option to stop further processing,
        or continuing

      Please see the OptionObjects class documentation for more information about Error Codes.
